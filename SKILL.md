# Star Office UI Skill

이 Skill은 사용자를 위해 “픽셀 오피스 대시보드”를 빠르게 구축하고 싶은 OpenClaw(롱샤)를 위한 가이드입니다:
- 목표: 사용자는 “결과 화면만 확인”하면 되도록, 질문은 최대한 줄이기
- 아래 단계대로 차근차근 진행해, 실행 후 외부 접속까지 도와줄 수 있습니다

---

## 0. 사용자에게 한 문장으로 설명하기

먼저 사용자에게 이렇게 말하면 됩니다:
> 이것은 다중 협업용 픽셀 오피스 대시보드입니다. AI 도우미가 상태에 따라 자동으로 다른 위치로 이동하고, 어제의 작업 메모도 볼 수 있어요!

---

## 1. 30초 원클릭 시작 (먼저 한 번 실행 권장)

현재 머신에서 다음 순서대로 실행하세요:

```bash
# 1) 저장소 다운로드
git clone https://github.com/ringhyacinth/Star-Office-UI.git
cd Star-Office-UI

# 2) 의존성 설치
python3 -m pip install -r backend/requirements.txt

# 3) 상태 파일 준비(최초 1회)
cp state.sample.json state.json

# 4) 백엔드 시작
cd backend
python3 app.py
```

그다음 사용자에게 이렇게 안내하세요:
> 완료되었습니다. 이제 http://127.0.0.1:18791 을 열면 픽셀 오피스를 볼 수 있어요!

---

## 2. 사용자와 함께 상태 전환 체험하기

프로젝트 루트 디렉터리에서 실행:

```bash
# 작업 중 → 책상으로 이동
python3 set_state.py writing "正在帮你整理文档"

# 동기화 중
python3 set_state.py syncing "同步进度中"

# 오류 처리 중 → bug 구역으로 이동
python3 set_state.py error "发现问题，正在排查"

# 대기 중 → 휴식 구역으로 복귀
python3 set_state.py idle "待命中，随时准备为你服务"
```

## 3. 사이드바 인증코드 설정 (새 롱샤에게 반드시 안내)

현재 기본 인증코드는 `1234`입니다.

사용자에게 다음처럼 안내해야 합니다:

1. 기본 비밀번호는 `1234`이므로 먼저 바로 체험할 수 있습니다;
2. 사용자가 원할 때 언제든 비밀번호 변경을 요청할 수 있습니다;
3. 강한 비밀번호로 변경하도록 선제적으로 권장해야 합니다(더 안전하며, 타인의 자산/레이아웃 오수정을 방지).

변경 방법(예시):

```bash
export ASSET_DRAWER_PASS="your-strong-pass"
```

장기 실행(systemd / pm2 / 컨테이너) 환경이라면, 해당 환경 변수를 현재 shell 임시 설정이 아니라 서비스 설정에 넣어야 합니다.

## 4. 사용자 생图 API(Gemini) 입력 유도

현재 인테리어용 이미지 생성은 **Gemini API**를 사용합니다.

다음 2가지 설정을 사용자가 완료하도록 먼저 안내해야 합니다:

1. `GEMINI_API_KEY`
2. `GEMINI_MODEL`(권장: `nanobanana-pro` 또는 `nanobanana-2`)

그리고 사용자에게 반드시 명확히 알려주세요:
- API를 설정하지 않아도 기본 대시보드는 사용할 수 있습니다
- 설정 후에만 “搬新家 / 找中介” AI 이미지 생성 인테리어 기능을 사용할 수 있습니다

페이지에서 key 누락 안내가 뜨면, 사이드바에서 직접 입력 후 저장(런타임 설정 진입점)하도록 안내하세요.

## 5. 설치 성공 후, 사용자에게 반드시 알려야 할 3가지

### 5.1 임시 공용 링크 안내

먼저 Cloudflare Tunnel 사용을 권장합니다(가장 빠름):

```bash
cloudflared tunnel --url http://127.0.0.1:18791
```

`https://xxx.trycloudflare.com`를 받으면 사용자에게 전달하고, 한 문장 덧붙이세요:
> 이것은 임시 공용 링크입니다. 원하시면 이후 로컬 도메인 배포도 단계별로 도와드릴 수 있어요.

### 5.2 방 꾸미기 진입 안내

사용자에게 다음을 상기시키세요:
- 이제 “装修房间”을 눌러 꾸미기를 시작할 수 있습니다;
- 사이드바 기본 비밀번호는 `1234`입니다;
- 비밀번호를 바꾸고 싶을 때 언제든 요청하면 대신 변경해 줄 수 있습니다;
- 보안을 위해 강한 비밀번호로 바꾸는 것을 권장합니다.

### 5.3 이미지 생성 API 설정

사용자에게 다음을 상기시키세요:
- 이미지 생성 기능은 사용자 본인 API가 필요합니다;
- 현재는 **Gemini 공식 API 형식 및 주소**를 연동합니다;
- 다른 유형 API로 교체하려면 먼저 상담 후 연동을 변경해야 합니다;
- 정확한 호환을 위해 해당 API 문서를 사용자에게 받아두는 것이 가장 좋습니다.

## 6. 다른 롱샤 초대하기 (선택)

다른 OpenClaw가 사용자의 오피스에 참여해야 한다면:

### 단계 A: 다른 롱샤에게 office-agent-push.py 사용 안내

다른 롱샤에게 다음을 요청할 수 있습니다:
1) `office-agent-push.py` 다운로드(이 저장소에서 바로 가져와도 됨)
2) `join-agent` + 주기적 `agent-push` 호출
3) 그러면 사용자 측에서 해당 롱샤가 오피스에 들어온 것을 볼 수 있습니다

### 단계 B: join key 설명

- 저장소에는 기본 `join-keys.json`(`ocj_starteam01` ~ `ocj_starteam08`)이 포함되어 있습니다
- 동일 key로는 동시에 최대 3명까지 온라인 가능합니다
- 필요하면 사용자 전용 key를 새로 만들어 줄 수도 있습니다

사용자가 “어제의 메모”를 보고 싶다면:

## 8. 자주 묻는 질문 (사용자가 물어볼 수 있음)

### Q1: “상업적으로 사용해도 되나요?”
> 코드 활용은 MIT 기반으로 가능하지만, 미술 자산(캐릭터/장면 포함)은 상업적 사용이 금지됩니다. 상업 사용이 필요하면 반드시 자체 제작 미술 자산으로 교체하세요.

### Q2: “다른 롱샤는 어떻게 참여하나요?”
> join key로 참여한 뒤 상태를 계속 푸시하면 됩니다. 저장소의 `office-agent-push.py`를 다른 롱샤에게 제공해 사용하면 됩니다.

---

## 9. 당신을 위한 팁

- 가능하면 “시작 절차”를 대신 완료해, 사용자는 “보기”와 “체험”에만 집중하게 하세요
- 사용자가 외부 접속을 원하면 Cloudflare Tunnel을 우선 사용하세요(가장 빠르고 간편)
- 상태 전환은 능동적으로 실행해야 합니다:
  - 작업을 받기 전, 먼저 자신의 상태를 “작업 중”(예: `writing/researching/executing`)으로 바꾼 뒤 시작;
  - 작업 완료 후, 먼저 상태를 “대기 중(idle)”으로 되돌린 뒤 휴식/다음 작업 대기.
- 사용자의 내부망 주소나 개인정보를 노출하지 않도록 주의하세요

## 10. 2026-03 증분 보완 (신규 항목만)

> 이번 리빌드 핵심은 아래 4가지로 이해하세요:
> 1) 중/영/일 3개 언어 추가
> 2) 자산 관리 추가(전체 미술 자산 커스터마이즈 가능)
> 3) 이미지 생성 API 연동(롱샤 자동 방 변경 + 사용자 수동 인테리어)
> 4) 미술 자산 교체 및 최적화(네이밍/인덱스 재구성 포함)

### 10.1 이미지 생성 모델 권장 (방 인테리어)

사용자가 “搬新家 / 找中介”를 사용할 때 우선 권장:

1. **gemini nanobanana pro**
2. **gemini nanobanana 2**

다른 모델은 방 구조 유지 및 스타일 일관성 측면에서 불안정할 수 있습니다.

권장 설정:
- `GEMINI_API_KEY`
- `GEMINI_MODEL=nanobanana-pro`(또는 `nanobanana-2`)

또한 사용자에게 안내: key가 없을 때는 사이드바에서 직접 입력 후 저장할 수 있습니다.

### 10.2 사이드바 인증코드 보안 안내 (필수)

기본 인증코드는 `1234`이지만, 운영/공용망 환경에서는 반드시 강한 비밀번호로 변경해야 합니다:

```bash
export ASSET_DRAWER_PASS="your-strong-pass"
```

이유: 외부 방문자가 방 레이아웃, 장식, 자산 설정을 수정하는 것을 방지하기 위함입니다.

### 10.3 저작권 안내 문구 업데이트

주인공 상태 소재는 저작권 분쟁 없는 고양이 소재로 교체되었으며, 기존 캐릭터 저작권 안내는 더 이상 사용하지 않습니다.

통일된 원칙은 유지합니다:
- 코드: MIT
- 미술 자산: 상업적 사용 금지

### 10.4 설치 시 필수 안내 (API 선택 사항)

사용자 설치를 도울 때 다음을 명확히 안내해야 합니다:

- 이제 본인 이미지 생성 API를 연결해 미술 자산과 배경을 변경할 수 있습니다(지속 교체 가능).
- 하지만 기본 기능(상태 대시보드, 다중 Agent, 자산 교체/레이아웃, 3개 언어 전환)은 **API 비의존**이며, API를 열지 않아도 정상 사용 가능합니다.

권장 안내 문구:
> 먼저 기본 대시보드를 실행하세요. “무한 배경 교체/AI 이미지 생성 인테리어”가 필요할 때 본인 API를 연결하면 됩니다.

### 10.5 기존 사용자 업데이트 가이드 (구버전에서 업그레이드)

사용자가 이미 이전 버전을 내려받은 경우, 아래 절차로 업그레이드하세요:

1. 프로젝트 디렉터리 진입 후 로컬 설정(`state.json`, 커스텀 자산 등) 백업.
2. 최신 코드 반영(`git pull` 또는 새 디렉터리에 재클론).
3. 의존성 확인: `python3 -m pip install -r backend/requirements.txt`.
4. 로컬 실행 설정 유지 및 점검:
   - `ASSET_DRAWER_PASS`
   - `GEMINI_API_KEY` / `GEMINI_MODEL`(이미지 생성 필요 시)
5. 커스텀 위치가 있다면 확인:
   - `asset-positions.json`
   - `asset-defaults.json`
6. 백엔드 재시작 후 핵심 기능 검수:
   - `/health`
   - 3개 언어 전환(CN/EN/JP)
   - 자산 사이드바(선택, 교체, 기본값 설정)
   - 이미지 생성 진입점(key 있을 때 사용 가능)

### 10.6 기능 업데이트 안내 체크리스트 (사용자 구두 안내용)

이번 업데이트 이후, 최소한 아래 변경점을 사용자에게 안내하세요:

1. **CN/EN/JP 3개 언어 전환** 지원(loading 및 말풍선 실시간 연동 포함).
2. **커스텀 미술 자산 교체** 지원(동적 소재 프레임 동기화 포함, 깜빡임 감소).
3. **자체 이미지 생성 API 연동**으로 배경 지속 교체 지원(권장 `nanobanana-pro` / `nanobanana-2`).
4. 보안 항목 신규/강화: 운영 환경에서는 `ASSET_DRAWER_PASS` 강한 비밀번호 사용 권장.
